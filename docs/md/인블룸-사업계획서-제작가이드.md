## 2. 브랜드 & 비전

### 2.1 브랜드 네이밍 의미

```
특별 = 평범하지 않은, 남다른 (일생에 한 번뿐인 순간)
시(時) = 시간, 순간 (결혼식, 경조사, 기념일)
Special Time = 글로벌 확장 가능
ㅌㅂㅅ = MZ세대 초성 문화 + 기억에 남는 발음
```

### 2.2 왜 웨딩에서 시작하는가?

1. **가장 높은 객단가**: 건당 평균 40만원 (본식영상 42.5만, 사진 45만)
2. **검증된 수요**: 연간 19만 쌍 결혼, 6조원 시장
3. **자연스러운 확장**: 웨딩 → 경조사 → 기업행사 (고객 생애주기)

## 4. 핵심 목표 수치

### 4.1 런칭 1년차 목표 (2026.06~12)

| 지표 | 목표 | 비고 |
|------|------|------|
| 등록 프리랜서 | **1,000명** | 핵심 KPI |
| MAU | **10,000명** | 수요자 기준 |
| 월 매출 | **650만원+** | 안정화 목표 |
| BEP 달성 | **11~12월** | 런칭 후 6~7개월 |

### 4.2 BEP 분석 (2단계)

**Phase 1 BEP (아카데미 기간, 1~5월):**
- 인건비 불가 → 순운영비 기준
- 프리랜서 200명 확보 목표

**Phase 2 BEP (예창패 선정 후, 6~12월):**
- **프리랜서 426명** 등록 시 고정비 BEP 달성
- 월 매출: 426명 × 17,384원 = 740만원

### 4.3 Unit Economics

| 지표 | 수치 | 근거 |
|------|------|------|
| Blended CAC | **44,000원** | 채널 믹스 기반 가중평균 |
| LTV | **352,000원** | 구독 티어별 가중평균 |

## 5. 차별화 기능

### 5.1 웨딩 크루 시스템

여러 프리랜서를 하나의 "크루"로 묶어 패키지 서비스 제공

| 크루 유형 | 구성 | 타깃 |
|----------|------|------|
| 필름라이크 크루 | 스냅+영상+축가+플로리스트 | 빈티지/감성 선호 |
| 페스티벌 크루 | DJ+플래시몹+네온사인 | 파티 무드 |
| 클래식 크루 | 현악4중주+합창단+클래식영상 | 전통 선호 |

### 5.2 웨딩 라운지 (3C Flywheel)

```
Content → Community → Commerce → Content...

[발견] 바이럴 웨딩 콘텐츠 탐색
[연결] 커뮤니티에서 프리랜서와 소통
[거래] 매칭 요청 및 예약
```

### 5.3 AI 매칭 로직

**MVP 단계 (LLM API 기반):**
1. 고객 취향 입력 (감성 키워드)
2. 포트폴리오 태그 자동 추출
3. 태그 기반 점수 매칭
4. Top 5 추천

**확장 단계:**
- 협업 필터링: "비슷한 취향 신부가 선택한 작가"
- 이미지 임베딩: 포트폴리오 시각적 유사도

### 5.4 AI 캐릭터 IP + 숏폼 콘텐츠

- 브랜드 캐릭터를 활용한 숏폼 콘텐츠 제작
- 틱톡/릴스 바이럴 전략
- 콘텐츠 → 유입 → 매칭 순환 구조

## 6. 민이형 피드백 적용

### 6.1 피드백 요약 (2025.12.08)

| 지적 사항 | 문제 | 해결 방향 |
|----------|------|----------|
| 팀원이 이해 못함 | 너무 복잡 | 초등학생 설명하듯 쉽게 |
| 숨고/크몽만 기억남 | 비교군 과다 | 비교군 최소화, 우리 이야기 80% |
| AI 매칭 부각 안됨 | 킬러피처 불명확 | 킬러콘텐츠로 전면 배치 |
| 수치 과함 | 30억, 3% | 보수적 수치 + 달성 로드맵 |
| 국익 관점 없음 | 정부 연계 부족 | 국가 문제 해결 프레이밍 |
| ESG 언급 | 4인 스타트업에 무의미 | 완전 삭제 |

### 6.2 수치 수정 (민이형 피드백 반영)

| 항목 | 기존 | 수정 |
|------|------|------|
| 3년차 매출 | ~~30억~~ | **10억** |
| 시장 점유율 | ~~3%~~ | **0.3%** |
| 1년차 프리랜서 | 유지 | **1,000명** |

### 6.3 국익 관점 프레이밍

```
[문제]
- 저출산 0.72명 (OECD 최하위)
- 결혼 비용 부담 → 결혼 기피
- 웨딩 피해 64% 증가 (2년간)

[해결]
"정부가 법으로 해결하려는 문제를 특별시가 플랫폼으로 먼저 해결"
- 공정위 18개 업체 시정 (2024.11)
- 깜깜이 스드메 방지법 발의 (2025.02)
```

## 7. 문서 작성 원칙

### 7.1 핵심 원칙

1. **쉽게 쓰기**: 팀원(비개발자)도 이해할 수 있는 수준
2. **우리 이야기 80%**: 비교군은 문제 제기용 10% 이하
3. **AI 매칭 전면 배치**: 첫 3페이지 내 킬러피처 설명
4. **수치에 근거**: 모든 숫자에 출처 명시
5. **국익 프레이밍**: 저출산/결혼기피 해결 강조

### 7.2 문서 분리 체계

| 문서 | 대상 | 분량 | 목적 |
|------|------|------|------|
| 예창패 사업계획서 | 심사위원 | 15~20p | 합격 |
| 팀원 가이드북 | 팀원 4명 | 10p | 역할 이해 |
| IR 피칭덱 | 투자자 | 10~15p | 투자/조언 |

### 7.3 예창패 사업계획서 구조 (권장)

```
1. 표지 (1p)
2. 목차 (1p)
3. Executive Summary (1p)
4. Problem - 문제 인식 (2~3p)
   - 국가적 문제 (저출산/결혼 기피)
   - 시장 문제 (피해 64%↑)
   - 고객 Pain Point
5. Solution - 해결책 (3~4p)
   - AI 매칭 (킬러피처!)
   - 웨딩 크루 시스템
   - 에스크로 결제
6. Market - 시장 분석 (2p)
   - TAM/SAM/SOM
   - 경쟁 환경 (최소화)
7. Business Model (2p)
   - 3중 수익 구조
   - Unit Economics
8. Scale-up - 성장 전략 (2p)
9. Team (1~2p)
10. Financial (1~2p)
11. Ask (1p)
```
