# PART 5. 팀 구성 & 역량

> 핵심 팀원 구성, 역할 분담, 역량 증빙 방법

---

## 핵심 팀 구성

### 팀 구조

```
CEO (대표)
├── CTO (기술 총괄)
└── CMO (마케팅/운영)
```

---

### CEO (대표)

| 항목 | 내용 |
|------|------|
| **역할** | 사업 총괄, IR, 전략 기획 |
| **핵심 역량** | 웨딩 업계 경험, 도메인 전문성 |
| **증빙** | 웨딩 관련 경력, 인터뷰 데이터 |

#### CEO 역량 포인트
- 웨딩 업계 5년 경력
- 프리랜서 네트워크 보유 (30명 인터뷰 완료)
- 직접 경험한 페인포인트 → 창업 동기

---

### CTO (기술 총괄)

| 항목 | 내용 |
|------|------|
| **역할** | 개발 총괄, AI/ML 모델, 아키텍처 |
| **핵심 역량** | 풀스택 개발, AI/ML 경험 |
| **증빙** | 포트폴리오, GitHub 프로필 |

#### CTO 역량 포인트
- 풀스택 개발 경력 N년
- AI/ML 프로젝트 경험
- 스타트업 개발 경험

---

### CMO (마케팅/운영)

| 항목 | 내용 |
|------|------|
| **역할** | 마케팅, 고객 확보, 운영 |
| **핵심 역량** | 퍼포먼스 마케팅, 커뮤니티 운영 |
| **증빙** | 마케팅 실적, 성과 데이터 |

#### CMO 역량 포인트
- 퍼포먼스 마케팅 경력
- SNS/커뮤니티 마케팅 경험
- 스타트업 그로스 경험

## 역량 증빙 방법

### 1. 도메인 전문성

| 증빙 방법 | 내용 |
|----------|------|
| **업계 경력** | 이력서, 경력 증명서 |
| **고객 인터뷰** | 인터뷰 녹취록, 분석 보고서 |
| **시장 분석** | 자체 조사 보고서 |
| **LoI (Letter of Intent)** | 프리랜서 등록 의향서 |

### 2. 기술 역량

| 증빙 방법 | 내용 |
|----------|------|
| **포트폴리오** | 이전 프로젝트 결과물 |
| **GitHub** | 오픈소스 기여, 코드 품질 |
| **MVP** | 실제 작동하는 프로토타입 |
| **기술 블로그** | 기술 글, 발표 자료 |

### 3. 실행력

| 증빙 방법 | 내용 |
|----------|------|
| **마일스톤 달성** | 계획 대비 실행 결과 |
| **Traction** | 사용자 수, 거래 건수 |
| **바이럴** | 미디어 노출, 커뮤니티 반응 |

## 평가위원이 보는 팀 포인트

### 긍정적 시그널

- ✅ 도메인 경험이 있는 대표
- ✅ 기술 팀이 이미 구성됨
- ✅ 각자 명확한 역할 분담
- ✅ 이전 협업 경험 (팀 케미스트리)
- ✅ 실제 고객/프리랜서 인터뷰 진행 (실행력)

### 부정적 시그널

- ❌ 모두 비즈니스 배경 (기술 인력 없음)
- ❌ 역할 중복/불명확
- ❌ 처음 만난 팀 (케미 검증 안 됨)
- ❌ 도메인 경험 전무
- ❌ 인터뷰/검증 없이 아이디어만

## 팀 스토리 (Why Us?)

### 창업 동기

> "직접 결혼 준비를 하면서 겪은 불편함이 창업의 시작이었습니다.
> 숨고에서 축가 작가를 찾는데 3주가 걸렸고,
> 가격 비교가 안 돼서 더 비싸게 계약한 적도 있었습니다.
> '웨딩 서브서비스만 전문으로 하는 플랫폼이 있으면 좋겠다'는 생각에서
> 특별시를 시작하게 되었습니다."

### 왜 우리 팀인가?

1. **도메인 전문성**: CEO의 웨딩 업계 경험 + 직접 경험한 페인포인트
2. **기술 역량**: CTO의 AI/ML 경험 + 풀스택 개발 능력
3. **실행력**: 이미 30명 인터뷰 완료, MVP 개발 착수
4. **팀 케미**: 이전 협업 경험으로 검증된 팀워크
